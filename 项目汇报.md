# 全栈开发项目汇报

## 一、项目概述

##后端启动命令
venv\Scripts\uvicorn app.main:app --reload --port 8000
### 1.1 项目简介
本项目是基于 **Next.js + FastAPI + MySQL** 的企业级员工管理系统，实现了用户管理、商品分类管理、商品管理三大核心模块，并完成了完整的登录认证体系。

### 1.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Next.js 16 (App Router) | React 框架，TypeScript 开发 |
| 前端 | TailwindCSS | 原子化 CSS 样式方案 |
| 前端 | axios | HTTP 请求库，自动 Token 携带 |
| 后端 | FastAPI | 高性能 Python Web 框架 |
| 后端 | SQLAlchemy | ORM 关系映射 |
| 后端 | PyJWT | JWT Token 生成与验证 |
| 后端 | bcrypt | 密码加密存储 |
| 数据库 | MySQL | 关系型数据库 |
| 数据库 | pymysql | Python MySQL 驱动 |

## 二、项目亮点

### 亮点一：清晰的分层架构设计

后端严格遵循 **四层架构** 设计，职责分离清晰：

```
请求 → Router(路由层) → Service(业务层) → Model(数据层) → 数据库
              ↓              ↓
         schemas(DTO)    业务逻辑处理
```

**优点**：
- 各层职责单一，便于维护和测试
- 数据校验在 schemas 层统一处理
- 业务逻辑在 service 层，与路由解耦

### 亮点二：JWT + bcrypt 完整认证体系

#### 密码安全存储（bcrypt）
```python
# 注册时：明文密码 → bcrypt 加密存储
hashed_password = get_password_hash(password)

# 登录时：用户输入密码 → bcrypt 比对
verify_password(plain_password, hashed_password)
```

#### JWT 无状态认证
```python
# 登录成功后生成 Token
access_token = create_access_token(data={"sub": admin.username, "admin_id": admin.id})

# 后续请求通过 Authorization Header 携带 Token
# Authorization: Bearer <token>
```

### 亮点三：JWT 守卫（AuthGuard）实现接口保护

通过 FastAPI 依赖注入实现全局接口保护：

```python
# dependencies/auth.py

    username = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM]).get("sub")
    # 2. 查询数据库验证管理员身份
    # 3. 验证失败抛出异常（401 未授权）
```

```python

        # 拦截所有请求，记录：
```

### 亮点五：统一响应格式 + 全局异常拦截

所有接口返回统一格式：

```json
{
  "code": 200,           // 状态码（200成功 / 400错误 / 401未授权 / 500异常）
  "message": "操作成功",  // 提示信息
  "data": {...}          // 业务数据
}
```

全局异常处理覆盖：
- ✅ Pydantic 数据校验失败
- ✅ SQLAlchemy 数据库异常
- ✅ JWT Token 过期/无效
- ✅ 自定义业务异常（如"分类下有商品，禁止删除"）
- ✅ 404 资源不存在

## 三、核心功能实现流程

### 3.1 用户登录流程

```
┌──────────┐     POST /api/auth/login      ┌─────────────┐
│  前端    │ ──────────────────────────→  │  后端       │
│  登录页  │                               │  AuthRouter │
└──────────┘                               └─────────────┘
      │                                          │
      │  1. 接收 {username, password}             │
      │  2. 查询 admins 表验证 bcrypt 密码        │
      │  3. 验证通过 → PyJWT 生成 Token           │
      │  4. 返回 {access_token}                   │
      ▼                                          ▼
┌──────────┐                              ┌─────────────┐
│ 保存到   │  ←───────────────────────────│ Token 响应  │
│ localStorage │                          └─────────────┘
└──────────┘
      │
      │  后续请求自动携带
      │  Authorization: Bearer <token>
      ▼
```

### 3.2 受保护接口访问流程

```
┌──────────┐     GET /api/users           ┌─────────────────┐
│  前端    │ ──────────────────────────→  │  AuthGuard       │
│  axios   │                             │  (JWT守卫)       │
└──────────┘                             └─────────────────┘
                                               │
                                               │  1. 从 Header 提取 Token
                                               │  2. PyJWT 解码验证
                                               │  3. 查询数据库确认身份
                                               ▼
                                         ┌─────────────┐
                                         │  验证通过   │
                                         │  → 执行路由  │
                                         │  → 返回数据  │
                                         └─────────────┘
```

### 3.3 商品分类删除流程（含业务校验）

```
┌────────────────────────────────────────────────────────────┐
│  删除分类时，必须先检查该分类下是否有商品                     │
└────────────────────────────────────────────────────────────┘

DELETE /api/categories/{id}

Service 层逻辑：
  1. 查询 products 表，统计该 category_id 的商品数量
  2. 如果 count > 0 → 抛出 BusinessException
     "该分类下有 X 个商品，禁止删除！请先删除相关商品"
  3. 如果 count = 0 → 执行 DELETE 操作

前端效果：
  - 有关联商品时，点击删除按钮弹出错误提示
  - 无关联商品时，弹出确认对话框，执行删除
```

### 3.4 前端 axios 请求拦截流程

```typescript
// lib/api.ts

// 请求拦截器：自动附加 Token
api.interceptors.request.use((config) => {
  const token = localStorage.getItem("token");
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器：统一处理 401 未授权
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem("token");
      window.location.href = "/login";  // Token 过期 → 跳转登录
    }
    return Promise.reject(error);
  }
);
```
