## 一、上线架构总览

```
用户浏览器
    │
    ▼  HTTPS :443 / HTTP :80
┌─────────────────────────────────────┐
│  Nginx（阿里云 ECS Ubuntu）          │
│  ├─ /        → Next.js :3000        │
│  └─ /api/*   → FastAPI :8000        │
└─────────────────────────────────────┘
    │                    │
    ▼                    ▼
 MySQL :3306         Python venv + uvicorn
 (本机 127.0.0.1)    (systemd 守护)
```

| 组件 | 端口 | 说明 |
|------|------|------|
| Nginx | 80 / 443 | 对外唯一入口，反向代理 |
| Next.js | 3000 | 仅本机监听，不对外开放 |
| FastAPI | 8000 | 仅本机监听，不对外开放 |
| MySQL | 3306 | 仅本机监听，不对外开放 |

**默认管理员账号**（首次启动自动创建）：`admin` / `admin123`  
上线后请**立即修改密码**（见阶段八后的安全加固）。

在动手前，请确认以下事项：

| 序号 | 准备项 | 说明 |
|------|--------|------|
| 1 | 阿里云 ECS 实例 | 建议 **2 核 4G** 起，系统盘 ≥ 40 GB |

## 三、部署前代码调整（必做）

> ✅ **已完成**：项目代码已支持环境变量，见 `frontend/.env.example`、`frontend/.env.production.example`、`backend/.env.example`。

在服务器部署时，复制示例文件并填入真实配置即可。本地开发**无需改动**，默认仍连 `localhost:8000`。

### 3.1 前端：API 地址改为环境变量

编辑 `frontend/lib/api.ts`，将 `baseURL` 改为：

```typescript
const api = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000",
  timeout: 10000,
  headers: {
    "Content-Type": "application/json",
  },
});
```

在 `frontend` 目录新建 `.env.production`（生产环境构建用）：

```env
# 有域名时（推荐）
NEXT_PUBLIC_API_URL=https://你的域名.com

# 仅用 IP 测试时（无 HTTPS）
# NEXT_PUBLIC_API_URL=http://你的公网IP
```

> **说明**：axios 请求路径已带 `/api/...`，所以 `NEXT_PUBLIC_API_URL` 只写域名或 IP，**不要**再加 `/api` 后缀。

### 3.2 后端：CORS 允许生产域名

编辑 `backend/app/main.py`，将 CORS 改为支持环境变量（推荐）：

```python
import os

# 逗号分隔多个来源，例如：
# CORS_ORIGINS=https://yourdomain.com,http://yourdomain.com
cors_origins = os.getenv(
    "CORS_ORIGINS",
    "http://localhost:3000"
).split(",")

app.add_middleware(
    CORSMiddleware,
    allow_origins=[o.strip() for o in cors_origins if o.strip()],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

在服务器 `backend/.env` 中配置：

```env
CORS_ORIGINS=https://你的域名.com,http://你的域名.com
```

若暂时只用 IP 访问：

```env
CORS_ORIGINS=http://你的公网IP
```

### 3.3 后端：生产环境 `.env` 模板

在 `backend` 目录准备 `.env`（**不要提交到 Git**），服务器上使用：

```env
# 数据库
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=fullstack
DB_PASSWORD=请替换为强密码
DB_NAME=fullstack_db

# JWT（务必更换为随机长字符串）
JWT_SECRET_KEY=请替换为至少32位随机字符串
JWT_ALGORITHM=HS256
JWT_EXPIRE_HOURS=2

# 应用
APP_HOST=0.0.0.0
APP_PORT=8000
DEBUG=False

# CORS
CORS_ORIGINS=https://你的域名.com
```

生成随机 JWT 密钥（在服务器执行）：

```bash
openssl rand -hex 32
```

### 3.4 验收（本地）

```powershell
# 前端
cd frontend
npm run build

# 后端确认无语法错误
cd backend
.\venv\Scripts\python.exe -c "from app.main import app; print('OK')"
```

## 四、阶段一：购买与初始化 ECS

### 4.1 购买 ECS

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com/)
2. 创建实例 → 选择 **Ubuntu 22.04 64 位**（或 24.04）
3. 规格建议：**2 vCPU / 4 GiB** 及以上
4. 网络：默认 VPC 即可，**分配公网 IP**
5. 登录方式二选一：
   - **密钥对**（推荐，更安全）
   - **自定义密码**（记住 root 密码）

### 4.2 配置安全组

进入 **网络与安全 → 安全组 → 配置规则 → 入方向**：

| 协议 | 端口 | 授权对象 | 说明 |
|------|------|----------|------|
| TCP | 22 | 你的办公 IP/32 | SSH，尽量限制来源 IP |
| TCP | 80 | 0.0.0.0/0 | HTTP |
| TCP | 443 | 0.0.0.0/0 | HTTPS |

> **不要**对公网开放 3000、8000、3306。

### 4.3 域名解析（有域名时）

在域名 DNS 控制台添加 **A 记录**：

```
主机记录: @ 或 www
记录值: 你的 ECS 公网 IP
```

等待解析生效（通常 5～30 分钟）。可用 `ping 你的域名.com` 验证。

**本阶段验收**：能通过 SSH 连上服务器（见下一阶段）。

## 八、阶段五：部署后端 FastAPI

### 8.1 上传代码到服务器

**方式 A：Git（推荐）**

```bash
mkdir -p /var/www
cd /var/www
git clone 你的仓库地址 fullstack
cd fullstack/backend
```

**方式 B：本地上传**

在本地用 WinSCP / `scp` 将整个项目上传到 `/var/www/fullstack`。

### 8.2 创建虚拟环境并安装依赖

```bash
cd /var/www/fullstack/backend
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

### 8.3 配置环境变量

```bash
vim /var/www/fullstack/backend/.env
```

粘贴 [第三节](#三部署前代码调整必做) 中的 `.env` 内容，并填入真实密码与 JWT 密钥。

### 8.4 首次启动（自动建表 + 创建管理员）

```bash
cd /var/www/fullstack/backend
source venv/bin/activate
uvicorn app.main:app --host 127.0.0.1 --port 8000
```

另开一个 SSH 窗口测试：

```bash
curl http://127.0.0.1:8000/
# 应返回 {"code":200,"message":"服务运行正常",...}
```

按 `Ctrl+C` 停止临时进程。

### 8.5（可选）填充测试数据

```bash
cd /var/www/fullstack/backend
source venv/bin/activate
python seed_data.py
```

### 8.6 配置 systemd 守护后端

```bash
vim /etc/systemd/system/fullstack-api.service
```

写入：

```ini
[Unit]
Description=Fullstack FastAPI
After=network.target mysql.service

[Service]
User=root
WorkingDirectory=/var/www/fullstack/backend
Environment="PATH=/var/www/fullstack/backend/venv/bin"
ExecStart=/var/www/fullstack/backend/venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动并设置开机自启：

```bash
systemctl daemon-reload
systemctl enable fullstack-api
systemctl start fullstack-api
systemctl status fullstack-api
```

**本阶段验收**：

```bash
curl http://127.0.0.1:8000/docs
# 有响应；systemctl status 为 active (running)
```

## 九、阶段六：部署前端 Next.js

### 9.1 安装依赖并构建

```bash
cd /var/www/fullstack/frontend

# 确认 .env.production 已配置 NEXT_PUBLIC_API_URL
cat .env.production

npm install
npm run build
```

构建成功后会生成 `.next` 目录。

### 9.2 配置 systemd 守护前端

```bash
vim /etc/systemd/system/fullstack-web.service
```

写入：

```ini
[Unit]
Description=Fullstack Next.js
After=network.target fullstack-api.service

[Service]
User=root
WorkingDirectory=/var/www/fullstack/frontend
Environment=NODE_ENV=production
Environment=PORT=3000
ExecStart=/usr/bin/npm run start -- -p 3000 -H 127.0.0.1
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

> 若 `npm run start` 不支持 `-H`，可改为：`ExecStart=/usr/bin/npx next start -p 3000 -H 127.0.0.1`

启动：

```bash
systemctl daemon-reload
systemctl enable fullstack-web
systemctl start fullstack-web
systemctl status fullstack-web
```

**本阶段验收**：

```bash
curl -I http://127.0.0.1:3000
# 返回 HTTP 200 或 307 跳转
```

## 十、阶段七：配置 Nginx 反向代理

### 10.1 创建站点配置

**有域名时**（将 `你的域名.com` 替换为真实域名）：

```bash
vim /etc/nginx/sites-available/fullstack
```

```nginx
server {
    listen 80;
    server_name 你的域名.com www.你的域名.com;

    client_max_body_size 10M;

    # 前端 Next.js
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 后端 FastAPI（路径已含 /api）
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Swagger 文档（生产环境可按需关闭）
    location /docs {
        proxy_pass http://127.0.0.1:8000/docs;
        proxy_set_header Host $host;
    }

    location /redoc {
        proxy_pass http://127.0.0.1:8000/redoc;
        proxy_set_header Host $host;
    }

    location /openapi.json {
        proxy_pass http://127.0.0.1:8000/openapi.json;
        proxy_set_header Host $host;
    }
}
```

**仅用 IP 访问时**，将 `server_name` 改为：

```nginx
server_name _;
```

### 10.2 启用配置

```bash
ln -sf /etc/nginx/sites-available/fullstack /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default   # 可选，避免与默认站冲突
nginx -t
systemctl reload nginx
```

**本阶段验收**：

- 浏览器打开 `http://你的域名.com` 或 `http://公网IP`，能看到登录页
- `http://你的域名.com/docs` 能打开 Swagger
- 使用 `admin` / `admin123` 能登录并操作各模块

## 十一、阶段八：配置 HTTPS（推荐）

有域名且已解析到 ECS 时执行。

### 11.1 安装 Certbot

```bash
apt install -y certbot python3-certbot-nginx
```

### 11.2 申请证书并自动配置 Nginx

```bash
certbot --nginx -d 你的域名.com -d www.你的域名.com
```

按提示输入邮箱、同意协议。成功后 Nginx 会自动配置 443。

### 11.3 自动续期

```bash
certbot renew --dry-run
```

### 11.4 更新前端 API 地址

若已启用 HTTPS，确保 `frontend/.env.production` 为：

```env
NEXT_PUBLIC_API_URL=https://你的域名.com
```

并重新构建、重启前端：

```bash
cd /var/www/fullstack/frontend
npm run build
systemctl restart fullstack-web
```

同时更新 `backend/.env` 中 `CORS_ORIGINS` 为 `https://你的域名.com`，然后：

```bash
systemctl restart fullstack-api
```

**本阶段验收**：浏览器访问 `https://你的域名.com`，地址栏有锁标志，登录与接口正常。

## 十四、常见问题排查

### 14.1 前端能打开，但登录报 Network Error

| 可能原因 | 处理 |
|----------|------|
| `NEXT_PUBLIC_API_URL` 未配置或错误 | 检查 `.env.production`，重新 `npm run build` 并重启 web |
| Nginx 未代理 `/api` | 检查 `location /api/` 配置，`nginx -t` 后 reload |
| CORS 未包含前端域名 | 检查 `backend/.env` 中 `CORS_ORIGINS` |

### 14.2 502 Bad Gateway

```bash
systemctl status fullstack-api fullstack-web
journalctl -u fullstack-api -n 50 --no-pager
journalctl -u fullstack-web -n 50 --no-pager
```

常见原因：后端未启动、前端构建失败、端口被占用。

### 14.3 数据库连接失败

```bash
# 检查 .env 中 DB 账号密码
mysql -u fullstack -p fullstack_db

# 查看后端日志
journalctl -u fullstack-api -n 100 --no-pager
```

### 14.4 修改代码后如何更新

```bash
cd /var/www/fullstack
git pull   # 或重新上传文件

# 后端
cd backend && source venv/bin/activate && pip install -r requirements.txt
systemctl restart fullstack-api

# 前端
cd ../frontend && npm install && npm run build
systemctl restart fullstack-web
```
